# Harness UI Tool — 폐쇄망 도메인 개발자용 바이브코딩 도구 계획

> 사내 폐쇄망 환경에서 도메인 개발자(금융·보험·계리 등)가 vLLM 모델을 활용해 안전한 바이브코딩을 수행하도록,
> 하네스·임베딩·라우팅 규칙을 직관적으로 관리·재사용하는 웹 UI 도구.
> 기준: 2026-06-24

---

## 0. 목표 한 줄

> **"AI 전문 지식 없이도 도메인 개발자가 vLLM 기반 하네스를 안전하게 쓰고, 규칙을 자신 있게 만들고 공유할 수 있게 한다."**

---

## 1. 사용 환경 가정 (제약 조건)

| 항목 | 가정 |
|---|---|
| 네트워크 | **폐쇄망** — 외부 API·SaaS·텔레메트리 전면 금지 |
| 모델 서빙 | 사내 **vLLM 인스턴스** (OpenAI 호환 엔드포인트) |
| 사용자 | 도메인 개발자 (금융·보험·계리 등). AI 비전문가. 코딩은 능숙 |
| 배포 | Docker 컨테이너, 사내 GitLab CI/CD |
| 인증 | 사내 LDAP/SSO (리버스 프록시로 위임) |
| 데이터 저장 | 로컬 파일 + Git (변경 이력) |

## 2. 사용자 페르소나 & 핵심 시나리오

### 페르소나
- **김개발 (도메인 개발자, 5년차)**
  - 금융 코어뱅킹 시스템 개발. Java/Spring 능숙.
  - LLM은 ChatGPT로 가끔 써본 정도. 임베딩·라우팅·벡터DB는 모름.
  - **하고 싶은 것**: "AI 어시스턴트 쓰는데, 위험한 금융 코드는 자동 검증되면 좋겠다"

### 핵심 시나리오
1. **온보딩**: 처음 도구 열고 5분 안에 사내 vLLM 연결 + 자기 도메인(금융) 선택 → 바로 바이브코딩 시작
2. **일상 코딩**: 코드 작성 요청 보냄 → 자동 라우팅 → 위험 작업이면 풀검증 → 결과 받음
3. **룰 추가**: 자기 팀에만 있는 비즈니스 룰(예: 신상품 한도) 추가 → 시험 → 저장
4. **공유**: 자기 팀의 룰을 옆 팀에 export → 옆 팀이 import → 즉시 사용

## 3. 핵심 기능 (5개 도메인 + 1개 메인)

```
┌──────────────────────────────────────────────────┐
│ 🏠 [홈]  요청을 보내고 결과 확인 (Playground)     │
├──────────────────────────────────────────────────┤
│ 📦 [규칙 가져오기/내보내기]   팀 간 공유          │
│ 🛡️ [안전 가드]                법령 상한·금지 패턴 │
│ 🎯 [작업 분류 규칙]            키워드·경로 매핑   │
│ 🤖 [AI 엔진 설정]              vLLM 연결           │
│ ⚙️ [고급]                       임계값·온톨로지   │
└──────────────────────────────────────────────────┘
```

### 3-1. 🏠 홈 — Playground (메인 화면)
**목적**: 도구가 무엇을 하는지 직접 보여줌. 학습·실험·일상 사용 통합.

```
[요청 입력]
  ┌─────────────────────────────────────────────┐
  │ 연체수수료 계산 함수를 추가하고 싶어요         │
  │ 이율 20%, ACT/365, HALF_UP                  │
  └─────────────────────────────────────────────┘
  
  [▶ 분석 시작]

[결과 (시각화)]
  ┌─ 1단계: 규칙 검사 ─────────────────────┐
  │  ✓ "연체수수료" 단어 매치 → 금융 작업    │
  │  → 풀검증 필요 (위험)                    │
  └──────────────────────────────────────────┘
  ┌─ 2단계: 안전 가드 ─────────────────────┐
  │  ✓ 이율 20% → 법정 상한 통과            │
  │  ✓ 금지 패턴 없음                       │
  └──────────────────────────────────────────┘
  ┌─ 3단계: 처리 계획 ─────────────────────┐
  │  → 6개 AI 에이전트가 협업              │
  │    (작성 · 리뷰 · 보안 · 규제 · 검증)  │
  │  → 예상 시간: 3~5분                    │
  │  → 예상 비용: 약 2000 토큰              │
  └──────────────────────────────────────────┘
  
  [그대로 진행] [수정]
```

**왜 메인인가**: 비AI 사용자가 "이 도구를 쓰면 무엇이 벌어지는가"를 즉시 이해. 학습 곡선 0.

### 3-2. 📦 규칙 가져오기/내보내기 — 공유의 핵심
**목적**: "우리가 만든 하네스/룰을 가져와서 쓰고, 자기 룰을 공유"

```
[가져오기]
  ZIP·tar 파일 업로드 또는 사내 Git 저장소 URL 입력
  ↓
  미리보기:
    📦 finance-pack-v1.2.0
       ├ 작업 분류 규칙 (3개 도메인, 38개 키워드)
       ├ 안전 가드 (8개 룰)
       ├ 라우팅 예시 (50개 문장)
       └ 금융 온톨로지 (8 개념, 6 법령)
  
  [내 환경과 충돌 5건]
    - "연체" 키워드: 이미 있음 → [덮어쓰기/유지/병합]
    - 가드 룰 ID 충돌 → [덮어쓰기/이름변경/스킵]
  
  [가져오기 진행]

[내보내기]
  ☑ 작업 분류 규칙
  ☑ 안전 가드
  ☐ 사내 전용 키워드 (export 제외)
  
  버전: 1.0.0 → [1.1.0 자동 제안]
  변경 요약: "신용카드 한도 룰 3건 추가"
  
  [ZIP 다운로드] [사내 Git에 푸시]
```

**왜 중요한가**:
- 사용자가 우리가 만든 finance pack을 **즉시 활용**
- 팀 간 룰 공유로 사내 표준화
- 버전 관리·rollback 가능

### 3-3. 🛡️ 안전 가드 — 비AI 친화 최상위
**목적**: 법령 상한·금지 패턴. 도메인 개발자가 가장 잘 이해할 수 있는 비즈니스 룰.

```
[법령 상한 룰]
  ┌──────────────────────────────────────────────┐
  │ 이자제한법 §2 · 대부업법                       │
  │ 연이율 상한: [   20  ]% ⓘ                    │
  │ 위반 시 동작: ⦿ 차단  ○ 경고  ○ 무시          │
  │                                              │
  │ [✓ 활성화]    [편집]    [삭제]               │
  └──────────────────────────────────────────────┘
  
  [+ 새 법령 상한 추가]

[금지 패턴]
  ┌──────────────────────────────────────────────┐
  │ 금액 변수에 부동소수점 금지                    │
  │ 패턴: float|double + amount|principal|fee    │
  │ 이유: 정밀도 손실 (BigDecimal 사용 필수)      │
  │ 위반 시: ⦿ 차단                              │
  └──────────────────────────────────────────────┘

[시험해보기]
  요청: [연 25% 이자 계산 함수 추가]
  결과: 🚫 차단 — "이자제한법 §2 상한(20%) 초과"
```

**비AI 사용자에게 가치**:
- "내 도메인의 법령 룰을 내가 직접 관리한다" — 익숙한 비즈니스 영역
- 시험 박스로 저장 전 동작 확인

## 6. 기술 스택

| 영역 | 선택 | 이유 |
|---|---|---|
| UI 프레임워크 | **Streamlit** | Python 단일 파일, 사내망 호환, 기존 스크립트 직접 호출 |
| 기존 로직 재사용 | `triage.py`, `check_guard.py`, ... | 그대로 사용 — UI는 얇은 래퍼 |
| 설정 검증 | `jsonschema`, `pydantic` | 저장 전 스키마 검증 |
| 시각화 | Streamlit native + Plotly | 라우팅 흐름·정확도 차트 |
| 인증 | 리버스 프록시 (Nginx + OIDC) 위임 | Streamlit 자체 인증 약함 |
| 변경 이력 | `gitpython` | 모든 룰 변경 자동 commit |
| 패키지 | Docker, docker-compose | 사내 표준 |
| 임포트/익스포트 | `tarfile`/`zipfile` + 매니페스트 JSON | 검증 + 충돌 감지 |

전부 **Apache-2.0 / MIT / BSD** — 사내망 사용 자유.

## 7. 단계별 개발 계획

### Phase 1 — MVP (3~4일)
**목표**: 폐쇄망 vLLM 연결 + 기본 플레이그라운드 + import

| 작업 | 산출물 |
|---|---|
| 프로젝트 골격 (Streamlit + Docker) | `Dockerfile`, `app.py`, `pages/` |
| 🤖 AI 엔진 설정 화면 | vLLM URL 입력·연결 테스트 |
| 🏠 Playground 화면 | 요청 입력 → triage·guard 호출 → 시각화 |
| 📦 가져오기 (import만) | ZIP 업로드 → 미리보기 → 적용 |
| 폐쇄망 모드 강제 (외부 호출 차단) | 환경변수 가드 |

**완료 기준**: 사내 PC에서 도커 띄우고, 사내 vLLM 연결 후 요청 입력 → 결과 표시.

### Phase 2 — 룰 편집 (3일)
**목표**: 비AI 사용자가 룰을 직접 만들 수 있게

| 작업 | 산출물 |
|---|---|
| 🛡️ 안전 가드 편집기 | 폼·시험 박스·저장 시 회귀 자동 실행 |
| 🎯 작업 분류 규칙 편집기 | 키워드/경로 폼 |
| 변경 이력 (Git commit auto) | 모든 저장 → commit + 메시지 자동 생성 |
| Rollback UI | 최근 10개 변경 목록 → 1클릭 복원 |

### Phase 3 — 공유·고급 (2일)
**목표**: 팀 간 공유, 고급 사용자 지원

| 작업 | 산출물 |
|---|---|
| 📦 내보내기 | 선택 항목 → ZIP + 매니페스트 |
| ⚙️ 고급 (라우팅 임계값·예시 문장·온톨로지) | 토글로 숨김 |
| 평가셋 실행·정확도 차트 | "내 룰의 정확도는?" |

### Phase 4 — 협업·운영 (선택, 추후)
**목표**: 사내 표준화

| 작업 | 산출물 |
|---|---|
| 룰 승인 워크플로 | PR 형식 — 검토자 지정·승인 |
| 감사 대시보드 | 누가 언제 무엇을 바꿨나 |
| 사용 통계 | 도메인별 풀검증 빈도·차단율 |

## 8. 보안·운영 고려

### 9-2. 온보딩 위저드 (첫 실행 5분)
